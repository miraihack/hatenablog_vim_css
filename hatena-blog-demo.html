Neovimのプラグイン管理をlazy.nvimに移行した

Neovim プラグイン Lua

経緯

これまでpacker.nvimを使っていたが、アーカイブされたのでlazy.nvimに移行することにした。移行作業自体は1時間程度で完了。

lazy.nvim は起動時間の最適化に優れており、遅延読み込みがデフォルトで有効になっている。

設定例

~/.config/nvim/lua/plugins/ ディレクトリにプラグインごとのファイルを配置する構成にした。

-- lua/plugins/telescope.lua
return {
  "nvim-telescope/telescope.nvim",
  tag = "0.1.8",
  dependencies = {
    "nvim-lua/plenary.nvim"
  },
  keys = {
    { "<leader>ff", "<cmd>Telescope find_files<cr>" },
    { "<leader>fg", "<cmd>Telescope live_grep<cr>" },
  },
}

移行で気をつけたこと

キーマッピングの遅延読み込み

keysテーブルを定義しておくと、そのキーが押されるまでプラグインが読み込まれない。起動時間が120ms → 45msに改善した。

依存関係の明示

dependenciesを正しく設定しないと、プラグインの読み込み順序で問題が起きることがある。

項目packer.nvimlazy.nvim
起動時間120ms45ms
遅延読み込み手動設定デフォルト
UIなし組み込み
ロックファイルなしあり

全体的に非常に満足。packerから移行を迷っている人には強くおすすめしたい。

Treesitterでシンタックスハイライトを強化する

NeovimTreesitter

nvim-treesitterとは

Treesitterはインクリメンタルなパーサーで、正規表現ベースのハイライトより遥かに正確なシンタックスハイライトを提供する。

  • 構文木ベースのハイライト
  • インデントの改善
  • テキストオブジェクトの拡張
  • コードフォールディング

インストール

return {
  "nvim-treesitter/nvim-treesitter",
  build = ":TSUpdate",
  config = function()
    require("nvim-treesitter.configs").setup({
      ensure_installed = {
        "lua", "vim", "vimdoc",
        "javascript", "typescript",
        "rust", "go", "python",
      },
      highlight = { enable = true },
      indent = { enable = true },
    })
  end,
}

特にRustとTypeScriptのハイライトが劇的に改善された。型情報やライフタイムの色分けが正確になった。

パフォーマンス

  1. 大きなファイル（1万行以上）でも高速
  2. 正規表現ベースより安定
  3. 言語ごとのパーサーが独立しているため、追加も簡単

正規表現ベースのsyntax onから移行して、もう戻れなくなった。

LSPの設定をnvim-lspconfigで一元管理する

NeovimLSP

なぜLSP？

Language Server Protocolを使うことで、エディタに依存しないコード補完・定義ジャンプ・リファクタリングが可能になる。

基本設定

local lspconfig = require("lspconfig")

-- Rust
lspconfig.rust_analyzer.setup({
  settings = {
    ["rust-analyzer"] = {
      checkOnSave = { command = "clippy" },
    },
  },
})

-- TypeScript
lspconfig.ts_ls.setup({})

-- Lua
lspconfig.lua_ls.setup({
  settings = {
    Lua = {
      diagnostics = { globals = { "vim" } },
    },
  },
})

キーマップ

キー動作
gd定義にジャンプ
Kホバー情報表示
<leader>rnリネーム
<leader>caコードアクション
gr参照一覧

IDEと遜色ないレベルの開発環境がターミナル内で実現できる。
